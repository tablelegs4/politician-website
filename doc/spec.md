# 政治家公式HP テンプレ技術仕様

## 1. 情報アーキテクチャ（ページ一覧）

### 必須ページ

| パス | 説明 |
|------|------|
| `/` | トップページ |
| `/policy/` | 政策一覧（重点 + 全件） |
| `/policy/[slug]/` | 政策詳細 |
| `/profile/` | プロフィール |
| `/contact/` | 相談・要望フォーム（外部フォームでも可） |
| `/privacy/` | プライバシーポリシー |

### 任意ページ

| パス | 説明 |
|------|------|
| `/archive/` | 議会資料・一般質問一覧（リンク集） |
| `/media/` | 掲載実績 |
| `/results/` | 実績 |
| `/support/` | 参加・協力（表現は慎重に） |

## 2. ヘッダー・共通UI仕様

### ヘッダー構成

- 左上ロゴ：**国民民主党（正式表記）**
- ヘッダー縦幅はやや広め
- 情報はやや大きめに表示

### 現職区分の表示

| 種別 | 表示例 |
|------|--------|
| 現職国会議員 | 参議院議員 / 衆議院議員 |
| 新人国会議員 | 〇〇県〇区 公認内定予定候補者 |
| 現職地方議員 | 〇〇市議会議員 |
| 新人地方議員 | 〇〇県〇〇市 公認内定予定候補者 |

### ナビゲーション

- メニューはTOPページ内のスクロール移動を基本とする
- ページ遷移が必要なメニュー（プライバシーポリシー等）には遷移アイコンを付与

## 3. TOPページ構成

### 3-1. ファーストビュー（1スクロール以内）

| 要素 | 仕様 |
|------|------|
| H1 | 立場が一瞬でわかる短いメッセージ（15〜25字） |
| サブ | 誰のために／何を改善するのか（2行） |
| 信頼要素 | 1〜3個（即時に見える箇所） |

**信頼要素の内容:**

- 現職：提案数、質問回数、対応件数など（更新日付き）
- 新人：専門領域、実務経験、地域活動実績など（検証可能な内容）

### 3-2. 重点政策ブロック

- 重点政策：**3つ**
- 各カード：見出し + 100〜140字
- 「政策根拠を見る」→ 政策詳細ページ
- 「政策一覧を見る」→ 政策一覧ページ

### 3-3. 伝えたいこと

- **300字 × 最大2本**
- 問いとセットで構成
- 想定Q:
  1. なぜ政治家を志したの？
  2. なぜ国民民主党を選んだの？

### 3-4. CTAセクション

**配置:** よくある質問の**上**

| 要素 | 内容 |
|------|------|
| H2 | 立場メッセージ（ファーストビューと同じ） |
| サブ | 誰のために／何を改善するのか |
| 一次CTA | 相談・要望を送る |
| 二次CTA | 活動レポートを受け取る |

### 3-5. FAQ

- FAQ：**3件**（誤解されやすい点を先回り）
- タイトルはカスタマイズ可（例：「山田たろうが答える！」）

### 3-6. 政策5択（任意）

- 余力があれば設置
- TOPに詰め込みすぎない

### 3-7. サイドバー（PCのみ）

- プロフィール（短）：写真 + 200字 + 「詳しく」リンク
- 相談・要望（再掲CTA）
- SNSフォロー

### 3-8. レスポンシブ対応

- 内容は同一。情報密度と配置のみ最適化
- SP：要約→展開（折りたたみ/詳細リンク）
- PC：2カラム可（右にプロフィールカード/CTA）

## 4. 政策ページ仕様

### 4-1. セクション構造（順序固定）

1. 課題（現状）
2. 提案（何をするか）
3. 進め方（ロードマップ：短期/中期/長期）
4. 目標
5. コスト
6. Q&A（反対意見含む）
7. 参考資料（最下部・外部リンク可）

### 4-2. 表現ルール

- 文章で書かない
- 箇条書き中心
- 2行以上OK
- 図解なし
- こくみんうさぎ使用
- 推奨分量：800〜1,200字

---

## 5. 活動報告仕様

### 5-1. カテゴリ

| カテゴリ | 内容 |
|----------|------|
| 議会 | 一般質問/委員会/提案/採択 |
| 地域 | 現場対応、意見交換会、視察 |
| 政策進捗 | ロードマップの更新 |
| メディア | 取材、掲載 |
| 地域課題の調査 | データ/ヒアリング |
| 提案 | 政策の素案、改善案 |
| 地域活動 | イベント、ボランティア等 |
| SNS誘導 | - |

### 5-2. 記事フォーマット

| 項目 | 必須 |
|------|------|
| タイトル | ○ |
| 公開日（更新日も） | ○ |
| カテゴリ | ○ |
| 要約（120〜200字） | ○ |
| 本文（短い段落） | ○ |
| 添付（写真/資料リンク） | - |
| 次の行動（相談/関連政策への導線） | - |

### 5-3. SNS・外部連携

- 外部SNSリンクは活動報告の場所とページフッターに配置
- X埋め込みはノイズになる可能性あり
- **note埋め込み推奨**

## 6. プロフィール仕様

### 6-1. TOPページ用プロフィール

- 詳細は別ページ
- 人柄が伝わる程度の情報量
- 選挙区表示:
  - 現職：「衆議院(〇期〇区：〇〇市、〇〇市、〇〇市)」
  - 新人：「〇〇県〇区：〇〇市、〇〇市、〇〇市」

### 6-2. プロフィール詳細ページ

**表現ルール:**

- 表組みは使わない
- title / content形式
- SPで崩れない設計

**基本情報フィールド:**

| フィールド | 必須 |
|------------|------|
| 写真 | ○ |
| 二つ名 | ○ |
| 氏名 | ○ |
| 生年月日（年のみ可） | ○ |
| 出身地 | - |
| 現住所 | - |
| 座右の銘 | ○ |
| 尊敬する政治家 | ○ |
| 家族構成 | - |
| 選挙区 | - |
| 小ネタ（1〜2個） | ○ |

**別枠:**

| フィールド | 必須 |
|------------|------|
| 経歴 | ○ |
| 政治信条（詳細） | ○ |
| 実績 | - |

## 7. FAQ仕様

### 7-1. タイトル

- カスタマイズ可（例：「山田たろうが答える！」）

### 7-2. テーマ

- 立場・価値観（短く）
- 財源/費用の考え方（断定より原則）
- 反対意見への回答（誠実に）
- よくある批判への説明

### 7-3. 想定質問例

- 他の政党と何が違うの？
- 財政に対してどう考えているの？
- ハイパーインフレを起こす政党？
- 国債発行は未来への借金？
- 労働組合の言いなりでは？

## 9. 実装要件（Astroテンプレ）

### 静的・軽量

- JSは原則ゼロ（必要なら局所）
- 画像最適化（WebP/AVIF推奨、lazy）
- CSSは単純に（global.css中心）

### コンテンツ管理（差し替え容易）

| パス | 内容 |
|------|------|
| `src/content/site/` | サイト基本情報（ロゴ、hero、CTA文言） |
| `src/content/profile/` | プロフィール |
| `src/content/policy/` | 政策（MD） |
| `src/content/activity/` | 活動（MD） |
| `public/images/` | 画像（差し替え対象を明示） |

---

## 10. GitHub Pages / Actions

### 原則

- `dist/` はコミットしない
- Actionsで build → deploy

### 公開形態

| 形態 | 設定 |
|------|------|
| Project Pages | `base: "/<repo>"` |
| User/Org Pages | base不要 |

---

## 11. 変更管理

- 文章・見出し・順番のルールを破らない
- コンテンツは `src/content` に集約
- 重要変更は docs/decisions.md に記録（任意）
