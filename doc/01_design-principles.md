# Design Principles
## 政治家HPテンプレの設計思想（最上位仕様）

## 1. 政治家HPの目的（Purpose）

### 目的

本サイトの目的は「閲覧者が政治家の活動・方針を理解し、信頼し、次の行動（相談・購読・参加・フォロー）に移る」こと。

**直接的な投票依頼（例：「私に投票してください」等）は行わない。**

## 2. 法令・運用上の前提

- 平時（選挙期間外）の政治活動サイトとして運用する
- 表現・導線は「情報提供」「相談受付」「活動報告」「方針説明」を中心にする
- 選挙期間対応（選挙モード）は本テンプレでは必須実装とせず、必要時は別途方針追加で対応する

---


## 3. 対象ユーザーと行動設計

### 想定ユーザー

| ユーザー層 | 期待する行動 |
| --- | --- |
| 地域住民（支持/未支持/無関心） | 政策理解、相談送信 |
| 支援者（後援会/ボランティア/紹介したい人） | 活動共有、参加 |
| メディア・関係者 | 実績確認、問い合わせ |
| 行政・議会関係者 | 提案内容、発言・資料確認 |

### 平時の推奨CTA（投票依頼に見えない）

| 優先度 | CTA |
| --- | --- |
| 一次CTA | 相談・要望を送る |
| 二次CTA | 活動レポートを受け取る（メール/LINE等） |
| 三次 | SNSフォロー |
| その他 | 活動実績の閲覧・共有 |

---

## 4. 表現ガイド（安全運用）

### 禁止/避ける表現（平時）

| 表現 | 理由 |
| --- | --- |
| 直接的な投票依頼 | 公職選挙法上のリスク |
| 対立を煽る断定（根拠なし） | 炎上リスク |
| 法解釈/医療断定（根拠なし） | 専門外の断定は危険 |

### 推奨表現

- 「〜を実現します」「〜を提案します」
- 「相談を受け付けています」
- 「活動報告」「実績」「提案内容」

---

## 5. テンプレ評価

### テンプレ（構造）の評価対象

テンプレとして評価されるのは、以下の要素である。

- 情報構造（ページ構成・導線）
- 各ページの役割定義
- メニューやコンテンツの提示順
- セクション構成および項目名
- 判断を補助するための仕組みや視点の提示

これらは **誰が使っても共通** であり、  
テンプレ自身が責任を持つべき領域である。

## 6. 自己決定を守るための原則

本テンプレでは、閲覧者の自己決定を尊重するため、以下を原則とする。

- 結論を押し付けない
- 断定的・煽動的な表現を多用しない
- できないこと・制約・トレードオフを明示する
- 判断材料を過不足なく提示する

HPの役割は「正解を示すこと」ではなく、  
**「判断に耐えうる情報を揃えること」**である。

---

## 7. 本ドキュメントの位置づけ

本 `design-principles.md` は、

- すべての仕様書（spec）
- すべてのページ設計
- すべてのコンテンツガイド
- すべての運用方針

に **優先する最上位文書** である。

以降の設計・実装・仕様追加は、  
本ドキュメントの原則と矛盾しないことを前提とする。

---

## 最終定義

本テンプレは、

> **政治家をよく見せるためのテンプレではない。  
> 閲覧者が後悔しない判断をするためのテンプレである。**

これが、すべての設計判断の出発点である。
