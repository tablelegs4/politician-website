# ドキュメント一覧

政治家公式HPテンプレートの設計・開発に関するドキュメントです。

---

## ドキュメント構成

| ファイル | 対象者 | 内容 |
|---------|--------|------|
| [spec-external.md](./spec-external.md) | クライアント | テンプレート概要・必要素材・納品物の説明 |
| [spec.md](./spec.md) | 開発者 | 技術仕様・ページ構成・コンポーネント仕様 |
| [design-principles.md](./design-principles.md) | 開発者 | 設計思想・デザイン原則・判断根拠 |
| [required-materials.md](./required-materials.md) | クライアント・開発者 | 画像・テキスト素材の詳細仕様 |
| [tasks.md](./tasks.md) | 開発者 | 実装タスクリスト・進捗管理 |

---

## 各ドキュメントの詳細

### spec-external.md（クライアント向け概要）

クライアントに渡す説明資料。以下を記載：

- サイトの目的
- ページ構成（必須/任意）
- 必要な素材・データの概要
- 納品物
- 公開前チェックリスト
- 運用上の注意点

### spec.md（技術仕様書）

開発者向けの詳細な技術仕様：

- 情報アーキテクチャ（ページ一覧・URL設計）
- ヘッダー・共通UI仕様
- TOPページ構成（ファーストビュー、政策、CTA、FAQ等）
- 政策ページ仕様（セクション構造、表現ルール）
- 活動報告仕様（カテゴリ、記事フォーマット）
- プロフィール仕様（足立形式）
- フッター・メニュー仕様
- 実装要件（Astro、コンテンツ管理）
- GitHub Pages / Actions設定
- 公開前チェックリスト

### design-principles.md（設計思想）

設計判断の根拠・デザイン原則：

- サイトの目的・法令上の前提
- 対象ユーザーと行動設計
- TOPページ構成の意図（なぜこの順序か）
- 政策・活動報告・プロフィール・FAQの設計思想
- 表現ガイド（安全運用）
- レスポンシブ設計の原則

### required-materials.md（素材仕様）

画像・テキスト素材の詳細な仕様：

- **画像素材**: ヒーロー画像、プロフィール写真、ロゴ、OGP、ファビコン、活動写真
  - サイズ・形式・構図の詳細指示
  - 撮影・編集のポイント
- **テキスト素材**: 基本情報、ヒーロー文言、信頼要素、プロフィール、政策、FAQ等
  - 文字数目安、入力例
- チェックリスト

### tasks.md（実装タスクリスト）

spec.mdとの比較に基づく実装タスク：

- **CRITICAL**: 必須対応（政策詳細、魂のメッセージ等）
- **HIGH**: 機能追加（プロフィール足立形式、FAQ更新等）
- **MEDIUM**: 構造改善（活動報告カテゴリ、政策MD構造等）
- **LOW**: 将来対応（テーマ名変更、管理機能等）
- データ構造の修正箇所
- 公開前チェックリスト
- 実装優先順位

---

## 更新ルール

1. 仕様変更は `spec.md` を先に更新
2. クライアント向け情報は `spec-external.md` に反映
3. 設計判断の根拠は `design-principles.md` に記録
4. 素材仕様の変更は `required-materials.md` を更新
5. 実装状況は `tasks.md` で管理
