# Specification

## 政治家HPテンプレ 構造仕様（Phase 2）

本ドキュメントは、
`design-principles.md` に定義された設計思想を前提として、
政治家HPテンプレの **情報構造・ページ構成・導線設計** を定義する。

本仕様は「何を書くか」や「どう表現するか」ではなく、
**どの情報を、どの順序で、どの役割として提示するか**を定めるものである。

---

## 1. サイト全体構造（Information Architecture）

本テンプレにおけるサイト構造は、
閲覧者が「判断を開始し、比較を整理し、次の行動を選択できる」
プロセスに沿って設計される。

### 基本構造

本サイトは**TOPページ（1ページ構成）**を中心とし、必要に応じて詳細ページへ遷移する構造とする。

| セクション       | 役割                             | 詳細ページ     |
| ---------------- | -------------------------------- | -------------- |
| 1. TOP全体       | 判断の起点・すべての情報への入口 | -              |
| 2. 政策          | 選択理由の理解                   | **あり**       |
| 3. 判断軸        | 判断基準の明示                   | なし           |
| 4. プロフィール  | 人物理解                         | **あり**       |
| 5. 活動・実績    | 行動の裏付け                     | なし（option） |
| 6. FAQ           | 懸念点・反対意見の整理           | なし           |
| 7. 行動          | 判断後の任意行動                 | なし           |

この順序は、
**説得や動員ではなく、判断補助を目的とした順序**であり、
TOPページ内のセクション配置およびナビゲーションに反映される。

---

## 2. メニュー設計方針

グローバルメニューは、以下の順序を基本とする。

1. TOP
2. 私のだいじな政策
3. 私の判断軸
4. プロフィール
5. 活動・実績
6. FAQ
7. 行動

### 設計意図

- 最初に「その基準で何を選んでいるか」を示す
- 次に「何を基準に判断する人物か」を提示する
- 最後に「行動喚起(CTA)」を配置する

TOPページはメニューには含めず、
常に判断の起点として扱う。

---

## 3. 役割定義（Page Roles）

### 3.1 TOPページ（各セクションの役割）

TOPページはすべてのコンテンツを1枚のWebページで網羅する。
以下は、TOPページ内の各セクションの役割・方針を定義する。

#### 3.1.1 TOP全体

- **役割**
  - AIDMAに則して視聴者に必要な情報を適切な順番で提示する
  - 必要な情報がどこにあるかを明確に提示する

- **方針**
  - 結論・主張・行動を急がせない
  - 詳細説明は行わない
  - ページ遷移可能な範囲で最低限の情報を提供する

---

#### 3.1.2 政策セクション（TOP内）

- **役割**
  - 閲覧者の関心と接続する入口
  - 判断を開始するきっかけを作る

- **方針**
  - 数を絞った重点テーマを提示する
  - 問題意識と方向性のみを示し、詳細は詳細ページへ誘導する
  - 解決策の詳細はここでは扱わない

---

#### 3.1.3 判断軸セクション（TOP内）

- **役割**
  - 本人が物事を判断する際の基準・優先順位を明示する
  - 明確な判断軸を示し、他の政治家との差別化を図る

- **方針**
  - 抽象論ではなく「判断に使われる基準」として記述する
  - 政策理解の補助として機能させる

---

#### 3.1.4 プロフィールセクション（TOP内）

- **役割**
  - 人物の概要を提示する
  - 詳細ページへの導線を提供する

- **方針**
  - 経歴の要約のみを提示する
  - 詳細な背景説明は詳細ページへ誘導する

---

#### 3.1.5 活動・実績セクション（TOP内・option）

新人議員などは実績が提示できないため、本セクションはオプションとする。

- **役割**
  - 方針や判断が、実際の行動として現れていることを示す
  - 言行一致の確認材料を提供する

- **方針**
  - 成果の誇示を目的としない
  - 判断軸と整合する実績を強調する

---

#### 3.1.6 FAQセクション（TOP内）

- **役割**
  - 懸念点・反対意見・誤解を公式に扱う
  - 懸念、疑念を早期に解消する
  - 反対意見を正直に載せて印象に残す（対決より解決）

- **方針**
  - 都合の良い質問のみを扱わない
  - 批判的視点やトレードオフを含める
  - 判断を先送りさせない構成とする

---

#### 3.1.7 行動セクション（TOP内）

- **役割**
  - 判断を終えた閲覧者が、任意で次の行動を選択できる場を提供する

- **想定行動**
  - 意見相談
  - 後援会
  - ボランティア
  - 寄付

- **方針**
  - 行動を強制しない
  - 判断未完了の状態での誘導を行わない
  - 行動は常に「結果」として位置づける

---

### 3.2 詳細ページの役割

詳細ページは、TOPページから遷移して深い情報を提供するページである。

#### 3.2.1 政策詳細ページ

- **役割**
  - この人物は何をする人なのか詳細に提示する
  - 課題に対する立場と選択理由を明確にする

- **方針**
  - 政策は「正解提示」ではなく「選択の提示」として扱う
  - 選ばなかった選択肢・制約条件を隠さない
  - 根拠を用いて説明を行い、信頼感を得る

---

#### 3.2.2 プロフィール詳細ページ

- **役割**
  - 判断軸が形成された背景を理解させる
  - 人物としての一貫性・文脈を提供する
  - 視聴者が共通点・共感点を発見できるようにする

- **方針**
  - 単なる経歴紹介に留めない
  - 価値観・経験・判断への影響を意識した構成とする

- **基本情報データ構造**

  | 項目 | フィールド名 | 必須 |
  | --- | --- | --- |
  | 氏名 | name | ○ |
  | 活動地域 | activityArea | ○ |
  | 生年 | birthYear | ○ |
  | 現住所 | currentAddress | ○ |
  | 出身地 | birthplace | ○ |
  | 最終学歴 | education | ○ |
  | 座右の銘 | motto | ○ |
  | 尊敬する人物 | respectedPerson | ○ |
  | 家族構成 | family | ○ |
  | 趣味 | hobbies | - |
  | 愛犬の名前 | petName | - |
  | 朝の日課 | morningRoutine | - |

---

## 4. 本仕様の適用範囲

本仕様は、

- 平時（選挙期間外）の政治活動サイト
- 個人HP・小規模サイト
- GitHub Pages 等の静的構成

を前提としている。

選挙期間対応や特別な運用が必要な場合は、
モードを切り替えることで実現する。(対応未定)

---

## 5. 本ドキュメントの位置づけ

本 `02_spec.md` は、

- ページ設計
- 情報構造
- 導線設計

に関する **基準文書** であり、
コンテンツ内容や表現方法は別ドキュメントで管理する。

本仕様は、
`design-principles.md` の思想と矛盾しないことを前提とする。
