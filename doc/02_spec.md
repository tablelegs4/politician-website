# Specification

## 政治家HPテンプレ 構造仕様（Phase 2）

本ドキュメントは、
`design-principles.md` に定義された設計思想を前提として、
政治家HPテンプレの **情報構造・ページ構成・導線設計** を定義する。

本仕様は「何を書くか」や「どう表現するか」ではなく、
**どの情報を、どの順序で、どの役割として提示するか**を定めるものである。

---

## 1. サイト全体構造（Information Architecture）

本テンプレにおけるサイト構造は、
閲覧者が「判断を開始し、比較を整理し、次の行動を選択できる」
プロセスに沿って設計される。

### 基本構造

本サイトは**TOPページ（1ページ構成）**を中心とし、必要に応じて詳細ページへ遷移する構造とする。

| セクション         | 役割                             | 詳細ページ     |
| ------------------ | -------------------------------- | -------------- |
| 1. TOP全体         | 判断の起点・すべての情報への入口 | -              |
| 2. 重点政策        | 選択理由の理解                   | **あり**       |
| 3. 魂のメッセージ  | 政治家の動機・原体験             | なし           |
| 4. プロフィール    | 人物理解                         | **あり**       |
| 5. 活動報告        | 行動の裏付け                     | あり（外部）   |
| 6. 相談誘導        | 判断後の任意行動（CTA）          | なし           |
| 7. FAQ             | 懸念点・反対意見の整理           | なし           |
| 8. 支援のお願い    | 応援者向け行動喚起               | なし           |

この順序は、
**説得や動員ではなく、判断補助を目的とした順序**であり、
TOPページ内のセクション配置およびナビゲーションに反映される。

---

## 2. メニュー設計方針

グローバルメニューは、PC版とSP版で異なる構成とする。

### PC版ナビゲーション（ページ遷移中心）

1. トップ
2. 魂のメッセージ（#message）
3. 政策（#policy）
4. 活動報告（#activity）
5. プロフィール（/profile/）
6. お問い合わせ（/contact/）
7. プライバシーポリシー（/privacy/）

### SP版ナビゲーション（アンカーリンク中心）

1. トップ（#top）
2. 魂のメッセージ（#message）
3. 政策（#policy）
4. プロフィール（#profile）
5. 活動報告（#activity）
6. よくある質問（#faq）
7. 応援する（#support）

### 設計意図

- **PC版**: ページ遷移とアンカーリンクを併用し、詳細ページへのアクセスを重視
- **SP版**: TOPページ内の全セクションへのアンカーリンクを優先し、1ページで完結する導線
- プライバシーポリシーやお問い合わせは、PC版でページ遷移として配置
- FAQと応援セクションは、SP版でアンカーリンクとして配置

## 3. 役割定義（Page Roles）

### 3.1 TOPページ（各セクションの役割）

TOPページはすべてのコンテンツを1枚のWebページで網羅する。
以下は、TOPページ内の各セクションの役割・方針を定義する。

#### 3.1.1 TOP全体

- **役割**
  - AIDMAに則して視聴者に必要な情報を適切な順番で提示する
  - 必要な情報がどこにあるかを明確に提示する

- **方針**
  - 結論・主張・行動を急がせない
  - 詳細説明は行わない
  - ページ遷移可能な範囲で最低限の情報を提供する

---

#### 3.1.2 政策セクション（TOP内）

- **役割**
  - 閲覧者の関心と接続する入口
  - 判断を開始するきっかけを作る

- **方針**
  - 数を絞った重点テーマを提示する
  - 問題意識と方向性のみを示し、詳細は詳細ページへ誘導する
  - 解決策の詳細はここでは扱わない

---

#### 3.1.3 魂のメッセージセクション（TOP内）

- **役割**
  - 政治家の動機・原体験を伝える
  - 「なぜ政治家を志したか」「なぜその政党を選んだか」などの根本的な問いに答える
  - 人物の価値観・信念を理解する起点となる

- **方針**
  - Q&A形式で2問程度（各300字以内）
  - 感情や体験を含む、人間味のある表現
  - アスタリスク（*）で囲んだ部分を強調表示

---

#### 3.1.4 プロフィールセクション（TOP内）

- **役割**
  - 人物の概要を提示する
  - 詳細ページへの導線を提供する
  - モバイルでは写真・名前・活動地域・SNSリンクを含む詳細表示

- **方針**
  - SP版: 写真、名前（ルビ付き）、活動地域、実績、SNSリンクを表示
  - PC版: サイドバーに配置（詳細はサイドバー仕様参照）
  - 詳細な背景説明は詳細ページへ誘導する

---

#### 3.1.5 活動報告セクション（TOP内）

- **役割**
  - 最新の活動を伝える
  - noteなど外部プラットフォームと連携し、最新情報を自動取得
  - 言行一致の確認材料を提供する

- **方針**
  - note RSSから最新5件を自動取得・表示
  - サムネイル、日付、タイトル、概要を表示
  - 外部リンクとして明示（アイコン表示）

---

#### 3.1.6 相談誘導セクション（TOP内）

- **役割**
  - 困りごとや相談を受け付ける導線を提供
  - 閲覧者に寄り添う姿勢を示す

- **方針**
  - キャッチコピー: 「お困りごと、ありませんか？」
  - プライマリCTA: お問い合わせフォームへ誘導
  - 個人情報保護の明示
  - コクサキャラクターで親しみやすさを演出

---

#### 3.1.7 FAQセクション（TOP内）

- **役割**
  - 国民民主党によくある質問に答える
  - 懸念点・反対意見・誤解を公式に扱う
  - 反対意見を正直に載せて印象に残す（対決より解決）

- **方針**
  - タイトル: 「{名前}が答える！国民民主党によくある質問！」
  - 都合の良い質問のみを扱わない
  - 批判的視点やトレードオフを含める
  - details/summary要素でアコーディオン形式
  - 各FAQに「リード」と「詳細」を含む

---

#### 3.1.8 支援のお願いセクション（TOP内）

- **役割**
  - 応援者向けの具体的な行動を提示する
  - 政治活動への参加方法を明示する

- **想定行動**
  - 党員・サポーターになる（外部リンク）
  - 後援会に入る
  - ボランティアで応援する
  - 寄付で応援する

- **方針**
  - カード型UIで4種類の応援方法を提示
  - 外部リンク（Google Formsなど）へ誘導
  - 行動を強制しない、選択肢を提示する形

---

### 3.2 詳細ページの役割

詳細ページは、TOPページから遷移して深い情報を提供するページである。

#### 3.2.1 政策詳細ページ

- **役割**
  - この人物は何をする人なのか詳細に提示する
  - 課題に対する立場と選択理由を明確にする

- **方針**
  - 政策は「正解提示」ではなく「選択の提示」として扱う
  - 選ばなかった選択肢・制約条件を隠さない
  - 根拠を用いて説明を行い、信頼感を得る

---

#### 3.2.2 プロフィール詳細ページ

- **役割**
  - 判断軸が形成された背景を理解させる
  - 人物としての一貫性・文脈を提供する
  - 視聴者が共通点・共感点を発見できるようにする

- **方針**
  - 単なる経歴紹介に留めない
  - 価値観・経験・判断への影響を意識した構成とする

- **基本情報データ構造**

  | 項目 | フィールド名 | 必須 |
  | --- | --- | --- |
  | 氏名 | name | ○ |
  | 活動地域 | activityArea | ○ |
  | 生年 | birthYear | ○ |
  | 現住所 | currentAddress | ○ |
  | 出身地 | birthplace | - |
  | 最終学歴 | education | ○ |
  | 座右の銘 | motto | ○ |
  | 尊敬する人物 | respectedPerson | ○ |
  | 家族構成 | family | ○ |
  | 趣味 | hobbies | - |
  | 愛犬の名前 | petName | - |
  | 朝の日課 | morningRoutine | - |

---

## 4. 本仕様の適用範囲

本仕様は、

- 平時（選挙期間外）の政治活動サイト
- 個人HP・小規模サイト
- GitHub Pages 等の静的構成

を前提としている。

選挙期間対応や特別な運用が必要な場合は、
モードを切り替えることで実現する。(対応未定)
