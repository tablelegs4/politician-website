# 政治家公式HP 設計思想・デザイン原則

## 1. サイトの目的

### 目的

本サイトの目的は「閲覧者が政治家の活動・方針を理解し、信頼し、次の行動（相談・購読・参加・フォロー）に移る」こと。

**直接的な投票依頼（例：「私に投票してください」等）は行わない。**

### 法令・運用上の前提

- 平時（選挙期間外）の政治活動サイトとして運用する
- 表現・導線は「情報提供」「相談受付」「活動報告」「方針説明」を中心にする
- 選挙期間対応（選挙モード）は本テンプレでは必須実装とせず、必要時は別途方針追加で対応する

---

## 2. 全体トーン

- 「政治・政策・日本」という言葉が持つ拒否反応を前提に設計
- 筆字などで「柔らかさ＋大事なこと」を同時に表現

## 4. TOPページ構成思想

### なぜこの順序か？

TOPページは「閲覧者が政治家に興味を持ってからCTAを提示する」設計。

```
1. ファーストビュー → 「誰か」を理解
2. 重点政策 → 「何をする人か」を理解
3. 伝えたいこと → 「なぜやるのか」を理解
4. 活動報告 → 「実際に動いている」を理解
5. CTA → ここで行動喚起
6. FAQ → 疑問・誤解の解消
```

### 各セクションの設計意図

#### ファーストビュー

- **目的:** 3秒で「誰のための政治家か」を伝える
- **注意:** CTAボタンを配置するレイアウトもあるが、これは「初見の閲覧者に興味を持たせる」目的であり、実際の行動喚起は中段以降で行う

#### 重点政策（3つ）

- **目的:** 具体的な活動内容を簡潔に伝える
- **なぜ3つ？:** 多すぎると散漫、少なすぎると薄い。3つが認知負荷と説得力のバランス

#### 伝えたいこと（300字×2本）

- **目的:** 動機・原体験を語り、人柄を伝える
- **設計:** 問いとセットで構成することで読みやすく
- **原則:** 人は論理より熱量に共感する

#### CTAセクション

- **配置:** FAQの**上**（重点政策・活動報告を見た後）
- **理由:** 十分な情報提供後に行動を促すことで、コンバージョン率向上

#### FAQ

- **目的:** 誤解されやすい点を先回りで解消
- **件数:** 3件（多すぎると読まれない）

## 5. 政策ページ設計思想

### 表現ルールの理由

| ルール | 理由 |
| --- | --- |
| 文章で書かない | 流し読みでも理解できるように |
| 箇条書き中心 | 情報の構造が一目でわかる |
| 図解なし | 更新コスト削減、一貫性維持 |
| こくみんうさぎ使用 | 親しみやすさ、党との一体感 |

### 政策の粒度

「政策の粒度」で悩まず、**政策説明テンプレとして統一**することで：

- 閲覧者が比較しやすい
- 執筆者が書きやすい
- 更新・メンテナンスが容易

---

## 6. 活動報告設計思想

### カテゴリ分けの意図

現職・新人両方に対応できるカテゴリ設計：

- **現職向け:** 議会、政策進捗
- **新人向け:** 地域課題の調査、提案、地域活動
- **共通:** 地域、メディア

### SNS連携の方針

| 方法 | 推奨度 | 理由 |
| --- | --- | --- |
| X埋め込み | △ | ノイズになる可能性あり |
| note埋め込み | ○ | 長文との相性が良い |
| SNSリンク | ○ | シンプル、軽量 |

## 7. プロフィール設計思想

### 出しすぎない理由（安全・炎上防止）

以下の情報は**掲載しない**：

| 情報 | リスク |
| --- | --- |
| 家族情報、子どもの学校など | プライバシー侵害、家族への攻撃 |
| 住所の過度な詳細 | 安全上の問題 |
| 詳細生年月日 | 不正なクレジットカード作成防止 |
| 支援者名簿的情報 | 個人情報保護 |

### 足立形式の意図

- **写真・二つ名・氏名:** 第一印象
- **座右の銘・尊敬する政治家:** 価値観の表明
- **小ネタ:** 親しみやすさ、人間味

---

## 8. FAQ設計思想

### 誤解の先回り

政治家への典型的な誤解・批判に対して、**攻撃的にならず誠実に**回答することで：

- 閲覧者の疑問を解消
- 「ちゃんと考えている人」という印象形成
- 炎上リスクの低減

### 回答の原則

- 断定より原則を示す
- 反対意見にも誠実に向き合う
- 人格攻撃にならない範囲で

---

## 9. 連絡・相談の設計思想

### 方針

- 投票依頼に見える導線は作らない
- 相談・要望・意見受付を中心にする
- **電話誘導はしない**（対応コスト、トラブル防止）
- **送りたくなる文言設計**

### ニックネーム可の理由

- 敷居を下げる
- 匿名でも意見を送りやすく
- 実名必須だと送信を躊躇する人が多い

---

## 10. 表現ガイド（安全運用）

### 禁止/避ける表現（平時）

| 表現 | 理由 |
| --- | --- |
| 直接的な投票依頼 | 公職選挙法上のリスク |
| 対立を煽る断定（根拠なし） | 炎上リスク |
| 法解釈/医療断定（根拠なし） | 専門外の断定は危険 |

### 推奨表現

- 「〜を実現します」「〜を提案します」
- 「相談を受け付けています」
- 「活動報告」「実績」「提案内容」

---

## 11. レスポンシブ設計の原則

### SP/PC共通

- **内容は同一**（情報格差を作らない）
- 情報密度と配置のみ最適化

### SP（スマートフォン）

- 要約→展開（折りたたみ/詳細リンク）
- 縦スクロール中心
- タップしやすいボタンサイズ

### PC

- 2カラム可（右にプロフィールカード/CTA）
- 情報密度を上げられる
- サイドバー活用

## 付録：設計判断の記録

### ページ遷移を極力しない方針

- **決定:** TOPページ内のスクロール移動を基本とする
- **理由:**
  - ページ遷移の度に離脱リスク
  - 情報の連続性が途切れる
  - SPでの操作性向上
- **例外:** プライバシーポリシー等、独立したページが適切なもの

### テーマ切り替えについて

- **現状:** 将来要件として保留
- **検討案:** 「頼れる」「柔らかい」の2テーマ
