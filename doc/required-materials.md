## 1. 画像素材

### 1-1. ヒーロー画像（TOP最重要）

| ファイル名 | `public/images/hero.jpg` |
|----------|--------------------------|
| **用途** | TOPページのファーストビュー背景画像 |
| **推奨サイズ** | 1920×1080px（16:9） |
| **ファイル形式** | JPG（WebP推奨）、500KB以下を目安 |
| **配置・構図** | 背景ベタ or 薄グラデ｜右：人物ピクトグラム（バストアップ）／左：H1・サブ・CTA余白 |

#### 詳細な撮影・編集指示

**レイアウト説明**
```
【PC表示】1920×1080px
┌────────────────────────────────────────────────────┐
│                                                    │
│  ┌──────────────────┐    ┌──────────────────┐     │
│  │                  │    │                  │     │
│  │  H1タイトル      │    │   人物画像       │     │
│  │  サブタイトル    │    │ （バストアップ） │     │
│  │  CTAボタン       │    │                  │     │
│  │                  │    │                  │     │
│  └──────────────────┘    └──────────────────┘     │
│                                                    │
│  【左側：テキスト余白】     【右側：人物配置】      │
└────────────────────────────────────────────────────┘
背景: ベタ色 or 薄いグラデーション
```

**撮影ポイント**
- **フレーミング**: バストアップ（頭頂部から胸元まで）
- **人物の向き**: 正面からやや斜め（10〜20度）、視線はカメラへ
- **表情**: 穏やかで親しみやすい微笑み
- **服装**: スーツ（ネイビーまたはチャコールグレー推奨）
- **背景**:
  - 推奨: 単色背景（白、グレー、薄いブルー）→ 切り抜きやすい
  - NG: ごちゃごちゃした背景
- **照明**: 順光（顔が明るく映るよう）
- **配置**: 画像右側に人物、左側にはテキスト用の余白を確保

**編集処理**
- 背景はベタ色または薄いグラデーションで統一
- 人物は画像右側に配置（ピクトグラム風）
- 左側はH1・サブタイトル・CTAボタン用の余白として確保
- 明るさ: やや明るめに補正
- 色温度: 自然な肌色を維持

### 1-2. プロフィール写真

| ファイル名 | `public/images/profile.jpg` |
|----------|----------------------------|
| **用途** | サイドバー、プロフィールページ、OGP画像 |
| **推奨サイズ** | 800×800px（1:1正方形） |
| **ファイル形式** | JPG（WebP推奨）、200KB以下を目安 |
| **配置・構図** | 中央寄せ｜人物ピクトグラム中心＋上下余白15〜20% |

#### 詳細な撮影・編集指示

**構図**
```
┌─────────────┐
│  余白15-20% │
│  ┌───────┐  │
│  │       │  │
│  │ 顔〜胸 │  │
│  │ の範囲 │  │
│  └───────┘  │
│  余白15-20% │
└─────────────┘
人物は中央寄せで配置
```

**撮影ポイント**

- **フレーミング**: 頭頂部から胸元まで（バストショット）
- **頭上の余白**: 画像上端から頭頂部まで15〜20%の余白
- **人物の向き**: 正面からやや斜め（10〜15度）
- **視線**: カメラを見る（親近感・信頼感）
- **表情**: 穏やかな微笑み
- **背景**: 単色（グレー、白、薄いブルーなど）または自然な屋外背景をぼかす
- **服装**: スーツ着用、ネクタイの色は落ち着いた色（紺、エンジなど）

**編集処理**

- 円形にトリミングされることを想定（顔が中央に来るよう配置）
- 肌補正は自然な範囲で
- 背景が複雑な場合はぼかしを強くする

### 1-6. 活動写真（汎用）

| ファイル名 | `public/images/activities/activity-01.jpg` など |
|----------|-----------------------------------------------|
| **用途** | 活動報告記事、TOPページのギャラリー |
| **推奨サイズ** | 1200×800px（3:2） |
| **ファイル形式** | JPG（WebP推奨）、各300KB以下 |
| **配置・構図** | 余白少なめ｜中央：人物＋活動要素（机・人・建物） |
| **必要枚数** | 最低3枚、推奨10枚以上 |

#### 撮影シーン別の詳細指示

**A. 議会活動（現職向け）**

- 議場での発言シーン（遠景〜中景）
- 委員会での質疑シーン
- 議場入口・庁舎前での立ち姿

**B. 地域活動**

- 住民との対話シーン（2〜3人での対話）
- 地域イベント参加（祭り、運動会など）
- 視察・現場確認（工事現場、施設など）

**C. 街頭活動**

- 駅前での挨拶活動（朝の様子）
- チラシ配布シーン
- 移動中・歩いている姿（動的な印象）

**D. オフィス・事務所**

- デスクワーク中（書類を読む、PCに向かう）
- スタッフとの打ち合わせ
- 来客対応シーン

**共通の撮影ポイント**

- 顔が明るく映ること
- 余白少なめで人物と活動要素を中央に配置
- 背景に個人情報が映り込まないよう注意
- 他の人物が映る場合は許可を得る
- 自然な動きのある瞬間を捉える（ポーズ写真は避ける）

### 2-2. ヒーロー文言（`src/content/site/` → hero）

| 項目 | 文字数目安 | 入力例 |
|-----|----------|-------|
| H1タイトル | 15〜25字 | 「地域とともに、未来をつくる」 |
| サブタイトル | 30〜50字 | 「すべての住民が安心して暮らせる社会の実現を目指して」 |

**H1タイトルの作成ガイド**
- 立場・姿勢が一瞬で伝わる
- 具体的すぎず、抽象的すぎない
- 「〜する」「〜をつくる」など動詞で終わると力強い
- 例：
  - 「暮らしに寄り添う政治を」
  - 「子どもの未来に全力を」
  - 「声を届ける、形にする」

### 2-7. 重点政策（`src/content/policy/` 3件必須）

各政策に必要な情報：

| 項目 | 文字数目安 | 備考 |
|-----|----------|------|
| タイトル | 10〜20字 | 具体的かつ端的に |
| 要約（summary） | 100〜140字 | TOP表示用 |
| カテゴリ | 2〜4字 | 「福祉」「教育」「防災」など |
| 本文 | 800〜1200字 | 下記構造に従う |

**政策ページの固定セクション（spec.md 4-1参照）**
1. 課題（現状）
2. 提案（何をするか）
3. 進め方（ロードマップ：短期/中期/長期）
4. 目標
5. コスト
6. Q&A（反対意見含む）
7. 参考資料（最下部・外部リンク可）

**表現ルール**
- 文章で書かない、箇条書き中心
- 図解なし
- こくみんうさぎ使用可

**政策タイトル例**
- 「保育園の待機児童ゼロを実現」
- 「高齢者の移動支援を拡充」
- 「災害に強いまちづくり」

### 2-8. FAQ（`src/content/site/` → faq、3件必須）

TOP表示用に3件、全体で5〜10件を推奨

**タイトル形式（カスタマイズ可）**
- 例：「山田たろうが答える！」

**FAQに入れるべきテーマ（spec.md 7-2参照）**
- 立場・価値観（短く）
- 財源/費用の考え方（断定より原則）
- 反対意見への回答（誠実に）
- よくある批判への説明（人格攻撃にならない範囲）

**想定質問例**
- 他の政党と何が違うの？
- 財政に対してどう考えているの？

## 4. 注意事項

### 画像に関して
- **著作権・肖像権**: 他者が映り込む写真は必ず許可を得る
- **個人情報**: 背景に住所・車ナンバーなどが映らないよう確認
- **ファイルサイズ**: 大きすぎると表示が遅くなるため最適化必須
- **差し替え前提**: 素材は後から差し替えられるよう、元データも保管

### テキストに関して
- **表記統一**: 氏名、役職名はサイト全体で統一
- **法令遵守**: 選挙期間中の表現には別途注意が必要
- **更新日付**: 数字を含む情報には更新日を明記
- **プレースホルダー残存**: 公開前に「サンプル」「〇〇」などの文言が残っていないか確認
